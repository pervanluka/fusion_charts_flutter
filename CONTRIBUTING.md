## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/fusion_charts_flutter.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Run tests: `flutter test`
6. Commit your changes: `git commit -m "Add your feature"`
7. Push to your fork: `git push origin feature/your-feature-name`
8. Open a Pull Request

## Development Setup

```bash
# Get dependencies
flutter pub get

# Run tests
flutter test

# Run example app
cd example && flutter run
```

## Code Guidelines

- Follow [Effective Dart](https://dart.dev/guides/language/effective-dart) style
- Add dartdoc comments for public APIs
- Write tests for new features
- Keep PRs focused on a single feature or fix

## Reporting Issues

- Use the [GitHub issue tracker](https://github.com/pervanluka/fusion_charts_flutter/issues)
- Include Flutter version (`flutter --version`)
- Provide a minimal reproducible example
